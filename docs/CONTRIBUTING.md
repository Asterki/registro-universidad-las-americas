# Contributing

Thank you for considering contributing. Please follow these guidelines:

1. Fork the repository and create feature branches for changes.
2. Write clear commit messages and open a pull request describing the change.
3. Update or add documentation in `docs/` when changing public behaviour or APIs.
4. Keep TypeScript types and shared contracts in `shared/` up to date.
5. Run and verify local tests (if present) before submitting a PR.

For major changes, open an issue first to discuss the design.
